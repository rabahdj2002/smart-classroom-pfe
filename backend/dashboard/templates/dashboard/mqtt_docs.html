3. Event Payload Contract
Identity And Timing
  • classroom_id or classroom_name or classroom
  • timestamp (optional ISO datetime)
RFID Keys
  • teacher_rfid or staff_rfid
  • student_rfid or student_rfids or students
Classroom Status Keys
  • occupied
  • lights_on
  • door
  • projector_on
  • smoke_detected
  • danger_indicator
Session Boundary Flags
  • new_session
  • session_start
  • start_new_session
  • force_new_session

Full event example

{
  "classroom_name": "d3",
  "timestamp": "2026-04-16T12:30:05+01:00",
  "teacher_rfid": "123456789123",
  "student_rfids": ["1234567891254", "STU-002"],
  "occupied": true,
  "lights_on": true,
  "door": false,
  "projector_on": true,
  "smoke_detected": false,
  "danger_indicator": false,
  "new_session": false
}
4. Session Lifecycle Rules
  • If no open session exists for the room, a new session is created from the event timestamp.
  • If a session boundary flag is true, any open session in that classroom is closed first.
  • If a different teacher starts in the same classroom, the previous open session is closed and a new one starts.
  • Students detected by RFID are linked directly to the active session.
  • Classroom occupancy, activity timestamp, and device status are updated from each event payload.
  • Auto-finish will generate reports for overdue sessions based on configured interval and duration.
5. Website Command Contract

The website publishes control messages to the room command topic.

Payload format

{
  "command": "lights" | "projector" | "smoke_reset",
  "classroom_id": 3,
  "classroom_name": "d3",
  "value": true | false
}

Behavior

  • lights with true or false toggles room lights.
  • projector with true or false toggles projector.
  • smoke_reset clears smoke and danger indicators after safety confirmation.

7. Security And Reliability Notes
  • Use broker authentication in production and avoid anonymous publish access.
  • Prefer stable classroom naming conventions so topic routing stays deterministic.
  • Send ISO timestamps when possible to keep report timing accurate across devices.
  • Do not publish retained event payloads unless intentionally replaying state.
  • If network drops, listener auto-reconnect attempts continue using configured reconnect delay.
8. Troubleshooting Checklist
  • No updates in dashboard: confirm broker host, port, and topic wildcard values above.
  • Unknown teacher/student warnings: verify RFID numbers match database entries exactly.
  • Session does not split when expected: send boundary flag new_session true or use a different teacher RFID.
  • Commands not executed by ESP: verify device subscribes to smartclass/classrooms/<room>/commands.
  • Unexpected occupancy state: include occupied value explicitly in each event payload.
